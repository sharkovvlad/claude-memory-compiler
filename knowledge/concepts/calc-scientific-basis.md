# Научное обоснование расчёта целей + консенсус v14

**Зачем статья:** единый источник-обоснование «почему мы считаем БЖУ именно так». Закрывает
повторяющийся вопрос владельца «это вообще научно?». Консенсус **подтверждён трижды**: мой аудит
+ внешний deep-research ИИ + клиническое слово владельца (2026-06-03). **Recent правит:** эта статья
новее раздела `personalized-macro-split §audit-v13` и при расхождении выигрывает.

## Каноничные стандарты (можно и нужно цитировать)

- **AMDR (Acceptable Macronutrient Distribution Ranges)** — IOM/NASEM Dietary Reference Intakes
  (подтверждено в DRI 2023): **углеводы 45–65%**, **жиры 20–35%**, **белок 10–35%** калорий.
  ВОЗ — близкие значения. Это и есть наш референс «диапазонов», а не точечных цифр.
- **ISSN (Int. Society of Sports Nutrition):** единственно правильного соотношения Ж/У **не
  существует** — при равных калориях и белке широкий спектр сплитов одинаково эффективен для
  композиции тела. Источник: ISSN position stands (PMC5470183, PMC5477153).
- **Белок:** ISSN база 1.4–2.0 г/кг массы; на дефиците выше — Helms 2014 (PMID 24092765) 2.3–3.1
  г/кг FFM, deficit-aware; Longland 2016 AJCN (2.4 vs 1.2 г/кг → +1.2 vs +0.1 кг мышц).
- **Жир-минимум:** ≥20% калорий / ≥0.8 г/кг для половых гормонов (Whittaker 2021 meta
  S0960076021000716: low-fat ↓ тестостерон SMD −0.38). Абсолютный физиологический минимум ~30 г/день.
- **Energy Availability:** ≥30 ккал/кг FFM/день (Loucks 2003 JCEM 88/1/297); ниже — RED-S.
- **PAL:** FAO/WHO/UNU 2004 + NASEM DRI 2023 — sedentary free-living ≈1.4 (не 1.2); vigorous 2.0–2.4.

## Ключевой вывод по жалобе «мало жира / много углеводов»

**Симптом верен, корень — не жир.** Грамм-floor жира 0.8 г/кг структурно ПРАВИЛЕН (на дефиците
потребность в жире не масштабируется с калориями → абсолютный грамм-floor лучше %-таргета). «Высокие
углеводы» — следствие того, что (а) белок не deficit-aware (занижен) и (б) жир прижат к нижней планке,
поэтому весь остаток уходит в углеводы. **Практическая ловушка:** белковые цельные продукты (мясо,
яйца, молочка) приходят С ЖИРОМ — выбить высокий белок при минимальном жире можно только обезжиренными
источниками + изолятами. Лечится **подъёмом ЦЕЛИ по жиру**, не снижением белка.

## Консенсус v14 (одобрено владельцем 2026-06-03, НЕ применено к проду)

Переход с «жир = floor + углеводы = остаток» на **жир = ТАРГЕТ + диапазоны**:

1. **Жир = таргет 1.0 г/кг фактич. веса** (кап 35% калорий, floor 0.8 г/кг). Был floor 0.8.
2. **Белок deficit-aware:** на `lose` коэффициент +~0.3 г/кг (или ×1.15). Внешний ИИ + Helms.
3. **Жир и углеводы — ДИАПАЗОНЫ (коридоры), не точки.** Белок — твёрдая цель/пол. Соответствует AMDR.
4. **PAL:** sedentary 1.2→1.3, light 1.375→1.45, moderate→1.6, heavy→1.8 + **Extreme-тир 2.0
   (cap 2.4)** для выносливости. EA-floor (≥30 ккал/кг FFM) — safety.
5. **ABW для obese — НЕ применять** (расхождение разрешено): внешний ИИ убедительно показал, что
   Mifflin на ФАКТИЧЕСКОМ весе уже точен у ожирения; ABW поверх → двойная коррекция/занижение.
   Вместо ABW — **кап ВЕСА ДЛЯ БЕЛКА** на уровне BMI~30 (чтобы 130кг не давали 260г белка). Калории —
   на фактическом весе. Моя исходная идея ABW снята.
6. **Adaptive TDEE, Diet Breaks — НЕ в SQL** (внешний ИИ: over-engineering для single-RPC); если
   делать — client-side / отдельный слой.

### Что внедрено (детали ниже — научное обоснование)

Второй внешний ИИ («Проверка расчётов КБЖУ диетологом») проверил **24 профиля v14** → в основном
подтвердил (жир-таргет 1.0 г/кг = «золотой стандарт», deficit-белок, RFM/Katch, maternal, teen-guard).
Нашёл 3 пункта; мой разбор — `Нутрициолог (аудит расчётов v13)/AUDIT_v15_external_review_response.md`
+ runnable-твин `_engine_v15.py` (20/24 профилей байт-в-байт = v14, меняются только 4 edge: 06/10/14/23).

1. **A — инверсия жиро-угл коридора у ожирения на дефиците (БАГ v14, ✅ фикс).** `0.8 г/кг × фактич.вес`
   превышает `35% × урезанные ккал` → жир-floor выше цели, углеводный коридор схлопывается. **Фикс:** для
   взрослых с ожирением (BMI≥30) **reference-вес (BMI 30)** как якорь и белка, и жира (floor/target/high).
   Калории — всегда на фактич.весе. Расширяет obese-cap из v14 #5 (там был только белок).
2. **B — «надбавку +0.3 г/кг только активным» → ❌ ОТКЛОНЕНО.** На дефиците белок = удержание массы+сытость,
   не гипертрофия (нужен и без тренировок); приверженность критична у sedentary. Кажущийся избыток у
   ожирения (P10 195г) лечится obese-cap (→138г), НЕ снятием надбавки. Надбавка остаётся универсальной.
3. **C — vegan-компаундинг (×1.25 поверх deficit+strength) → 🟡 ЧАСТИЧНО.** Два потолка белка: научный
   `min(3.1 г/кг FFM, 2.5 г/кг ref)` (Helms) + **мягкий vegan-cap 2.2 г/кг факт.веса** (owner-выбор,
   ЖКТ-переносимость). P23 229→187г, P14 230→176г.

**Durable:** прод = **v13**. Порядок миграций: v14 ядро (deficit-белок первым) → v15-дельты (ref-вес
ожирения + потолки белка), последовательно, база из `pg_get_functiondef` живого прода. v15 трогает 0
реальных юзеров сейчас. golden-кейсы: 06/10/14/23.

## Диеты и осознанный выбор (направление, дизайн не закрыт)

- **Научно обоснованные паттерны:** средиземноморская (сильнейшие данные — CVD/долголетие), DASH,
  сбалансированный дефицит, высокобелковая, low-carb/keto (валидна для похудения и спец-состояний,
  НЕ магия), plant-based/vegan, интервальное голодание/TRE (инструмент приверженности, эквивалент
  обычному дефициту). **Различаются выбором продуктов и ТАЙМИНГОМ больше, чем нашей макро-математикой.**
- **Что уже учтено:** цель (lose/gain/maintain) + diet_type (omnivore/veg/vegan). **Не учтено:**
  макро-ПАТТЕРН (keto/low-carb/high-carb-endurance) и ТАЙМИНГ (IF/TRE).
- **Keto-режим требует ослабить углеводный floor** (owner 50 / female 100) + warning по щитовидке/циклу.
- **Принцип:** keto/IF — выбираемые предпочтения с safety-обвязкой + disclaimer. Продлённое голодание
  (24ч+) активно НЕ рекомендуем (РПП-риск, anti-shame позиционирование), но осознанный выбор уважаем.

## Покрытие: 20 профилей — это НЕ 20 «средних» юзеров

Формула **параметрическая**: уникальный результат на каждую комбинацию (пол/возраст/вес/рост/
активность/тренировки/цель/скорость/фенотип/диета/талия/maternal) — тысячи различных выходов, не 20
бакетов. 20 профилей — это **тест-кейсы для проверки веток кода**, не вселенная юзеров. Реальные
юзеры получают индивидуальные числа. Ограничение любой формулы: она ПРЕДИКТИВНА (±10–15% межличностная
вариация) → истинная персонализация = Adaptive TDEE по тренду веса (P3, будущее). Сейчас даём
индивидуальный СТАРТ; калибровка по факту — следующий рубеж.
