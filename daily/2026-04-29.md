## Session: Migration 147 + Python is_inline handover (Back navigation overhaul)

**Контекст.** Закрытие двух связанных багов навигации + архитектурный pivot для cmd_back: переход от tree-walk (back_default) к path-walk (nav_stack как реальный history). Migrations 146 + 147 применены DB-агентом, Python handover закоммичен (`eb3c93d`), n8n side-patch — параллельно отдельным subagent'ом.

### Что починили

- **Bug «Назад иногда уводит в Профиль»** — root cause: hardcoded fallback `profile_main` в `process_user_input` Priority 3 (когда stack пуст и back_default NULL). Migration 146 (`146_back_fallback_stats_main_root.sql`) переключает Priority 3 fallback на `stats_main` — корневой экран бота, к которому естественно возвращаться при потере контекста.
- **Bug «Назад из Квестов не сработала с 1-го раза»** — root cause был в Answer Callback Query placement (inline-кнопка "зависала" пока не отработает основной flow). Починен сегодня ACQ chip-агентом (отдельная сессия) — ACQ вынесен в параллельную ветку перед основным processing.
- **Глубокая история навигации** — Migration 147 (`147_back_i_came_from_navigation.sql`) полностью переработал cmd_back:
  - `push_nav` теперь **truncate-on-existing**: если destination уже в стеке, обрезаем хвост вместо дублирования (защита от циклов A→B→C→B).
  - `advisory_xact_lock` на `users.id` — защита от concurrent agents, дёргающих стек одновременно.
  - **Input-source-aware fastpath** через `p_cb_context->>'is_inline'`: default TRUE (push в стек, inline navigation), FALSE (wipe stack, reply-kb «I came from elsewhere» semantics).

### Архитектурное решение

**Не делать хардкод-список Reply KB callbacks в SQL.** Вместо этого вызывающая сторона (Python proxy / n8n) передаёт `is_inline` в JSONB cb_context. SQL применяет default TRUE для backward compat — старые callers продолжают работать как inline.

**Семантика:**
- `is_inline=TRUE` (или omit) — обычный inline tap (callback_query из inline-keyboard) → push destination в стек.
- `is_inline=FALSE` — Reply KB tap (synth callback из reply-keyboard) → **wipe** стек и push fresh root → следующий Back уйдёт в Priority 3 fallback (stats_main).

Когда новые Reply KB icons появятся — обновляется **только** `dispatcher/router.py`, без SQL миграций.
