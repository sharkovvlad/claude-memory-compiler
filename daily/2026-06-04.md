### 🔑 Durable gotcha — РЕНДЕР `add_food_prompt` = легаси n8n `04_Menu`, читает ключ НАПРЯМУЮ
- Reply-кнопка `icon_add_food` в `dispatcher/router.py` (~860) **проваливается** в секцию 4l → `target="menu"` → `forward_to_n8n` (нет Python-обработчика «menu»; menu_v3 ≠ menu).
- Ключ `progress.add_food_prompt` **не имеет** text_key-референсов ни в Python, ни в ui_screens, ни в n8n_workflows/*.json репо. GET live `04_Menu` (id `JQsipPWxijse3F0b`) показал ноду с `text: $json.progress.add_food_prompt || "📸 Send a photo..."`, **`parse_mode: "HTML"`**.
- **2 следствия для любой правки этого ключа:**
  1. **Формат = HTML** (`<b>`/`<i>`/`<code>`), НЕ Markdown. `**bold**` отрендерится буквально. (Номс изначально предложил Markdown — поймали на ревью.)
  2. **Scalar, НЕ JSONB-array.** Нода берёт ключ как строку без random-pick (в отличие от Dumb Renderer в `03_AI_Engine`/errors, который делает `Math.random` по массиву). JSONB-array → в Telegram `text` уедет JS-массив → битое сообщение. Вариативность (dialog-variants) станет возможна только когда add_food переедет в Python (food_log handler) — тогда `_resolve_text` сам делает `random.choice`.
- Хардкоженный EN-fallback в ноде (`📸 Send a photo or type what you ate`) теперь стейл, но безвреден (срабатывает только если ключ NULL — не бывает). Легаси `04_Menu` усыхает, не трогал.
