### 2. Late-night close META

Owner-mandated cutoff `local_hour >= 22`. Нутрициологическое обоснование — **Sutton 2018, Chow 2020, NHANES Sun 2021**: energy intake ≥22:00 ассоциируется с худшим HbA1c/BMI; meals <2h до сна → fragmented sleep + GERD risk. Для среднего bed-time 23:30-00:30 — h≥22 правильный default cutoff. Мы **не** говорим «не ешь» (РПП-safety), мы перестаём **пушить**.

```python
def _late_night_close_meta(local_hour, *, day_status, protein_pct, kcal_pct):
    if local_hour < 22:
        return None
    soft_eligible = (
        day_status in ("severe_deficit", "under_target_evening")
        and protein_pct_f < 60.0 and kcal_pct_f < 80.0
    )
    if soft_eligible:
        return (SOFT_TEXT, "late_night_soft")   # «пара ложек творога» допустимо
    return (HARD_TEXT, "late_night_close")       # next_meal_suggested=false принудительно
```

Гейт за `app_constants.sage_late_night_close_enabled` (default `true`). food_log surface не имеет `day_status` → всегда HARD CLOSE. my_day имеет → SOFT branch доступен.

## Lessons (для KB compiler'a)

1. **Few-shot voice cards beats N+1-я META** для tone control. После 6 МETA-итераций emotion=smirk вырос (80→100%) — кризис «curse of instructions». Эталоны в начале промпта снимают это быстрее чем правила в конце.
2. **functional-claim filter** — новое правило для Sage tone review: любое «X поддержит Y» / «X улучшает Z» — псевдомедицинский claim, нарушает FORBIDDEN secrtion. Я флагнул на эталоне Номс-агента «творог поддержит мышцы и ЦНС».
3. **«лишних калорий» = anti-shame trigger** — слово «лишний» в FORBIDDEN-секции уже фактически было («перебор», «too much fat»), но не явно перечислено. Эталон Номс-агента включал это — поправил.
4. **psевдонаучные «системы»** — «метаболическая система» не существует как термин. Sage scientist part обязан использовать настоящие термины (метаболизм, обмен веществ) или не использовать никакие.
5. **i18n misconception** — voice cards в system prompt НЕ требуют per-language миграций. Single-source pattern (mig 412) — output language как payload параметр. Каждый раз при tone-of-voice работе нужно явно пояснять owner'у.

## Вечер 06-13 (поздняя сессия) — Vision CULTURE GUARD push-back + tiebreaker tightening

### Запрос и push-back

Brief: добавить «CULTURE GUARD» в vision food-recognition prompt — при ambiguous фото для AR/FA/ID → halal-safe default (chicken/beef вместо pork); для HI → vegetarian-default. Аналогично PR #392/#394 для Sage.

**Push-back перед кодом.** Проверил [services/ai_recognition.py:460-489](services/ai_recognition.py:460) — `_build_location_hint` уже передаёт country и формулировка специально дисциплинированная: «PRIMARY evidence = photo; country ONLY to break ties when name is ambiguous; never override visual cues with country prior». Предложенный halal-default прямо ломает этот контракт:

- Pork shoulder vs beef brisket **визуально различимы** (фактура, цвет, прожилки). Default к beef для AR-юзера = неверные калории (~30-40% дельта). Бот, который врёт «из доброты», = сломанный трекер.

- Sage CULTURE GUARD ≠ vision CULTURE GUARD. Sage — **тон персонажа** (субъективное). Vision — **фактическая классификация**. Накладывать «культурную линзу» на факт = data-quality regression.

Owner выбрал «strengthen existing tiebreaker» (узкий патч без halal-bias) из 3 опций.
