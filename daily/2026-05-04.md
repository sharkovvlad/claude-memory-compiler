## Миграция 167 — фикс ambiguity в cron_check_streak_breaks (Агент 3)

**Root cause:** копи-пастный регресс через две последовательные миграции:

**Фикс в 167:** объединяет фикс 036 + оба фильтра (`is_bot=false` от 042, `deleted_at IS NULL` от 166):
- minimal `SELECT u.telegram_id` в обоих CTE,
- qualified `SET streak_freezes = u.streak_freezes - 1`,
- `is_bot = false` × 2 + `deleted_at IS NULL` × 2 (оба CTE).

**Apply:** psycopg2 → applied + smoke `SELECT cron_check_streak_breaks()` вернул `{frozen:[], broken:[]}` (0 кандидатов в момент запуска — норма для UTC 13:30, локальная полночь только у узкого пояса).

**Команда (atomic, `BEGIN; ... COMMIT;`):**
```sql

BEGIN;
DELETE FROM workflow_published_version WHERE workflowId IN (...);  -- 1 row
DELETE FROM workflow_entity WHERE id IN (...);                      -- 5 rows
COMMIT;
```

## TON checker NoneType + централизация admin alerts в BaseCron

**Симптом:** `TonPaymentCheckerCron` падал каждые 5 минут (apscheduler репортил «executed successfully», в логах `[TonPaymentChecker] FAILED 0.53s` + stack trace). Stack:

```
File "crons/ton_payment_checker.py", line 104, in _process_transaction
    comment = tx.get("comment", "") or tx.get("forward_payload", {}).get("value", "")
AttributeError: 'NoneType' object has no attribute 'get'
```

**Root cause:** `dict.get("k", default)` возвращает `default` только если ключа НЕТ. Если ключ присутствует со значением `None` — а TON API так и отдаёт `forward_payload: null` для multi-hop transfers — возвращается `None`, и `.get("value", "")` падает.

**Фикс в `crons/ton_payment_checker.py:103-108`:**
```python
fp = tx.get("forward_payload") or {}
comment = tx.get("comment", "") or fp.get("value", "")
```

**Параллельно — централизация admin alerts в `BaseCron`:** lesson из 04.05 (silent cron failure pattern, `cron_check_streak_breaks` сутки падал незамеченным) был частично применён только в `claude/adoring-ardinghelli-902b86` (file-local helper в `streak_checker.py`), но в main не смержен. Вместо копипасты по всем 11 cron-jobs — вынес в `crons/base.py`:
