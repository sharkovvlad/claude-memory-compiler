### Ключевой инсайт: cache на уровне Node UUID

До сегодня считалось что cache привязан к workflow ID. После Variant B ясно:

**Runtime cache n8n Cloud привязан к UUID ноды Switch (`menu-router-5690fc82`), не к workflow ID.**

При POST клонирования UUIDs нод **сохраняются** (node-level `id` поля копируются 1:1) → новый workflow получает тот же badly-cached Switch state. Новый workflow_id не помогает.
