## Bugfix: Reply-кнопки не работают (callback_message_id fallback "0")

**Симптом:** Reply-кнопки (☀️ Мой день, 👤 Профиль, 🚀 Прогресс) показывают "typing..." но ответ не приходит.

**Корневая причина:** Template Engine v2.1 (выше в этом же логе) установил `_cb_msg_id` fallback = `0` вместо `''`.
- Для reply-кнопок (нет callback_query) → `_cb_msg_id = 0`
- Dispatcher Set node (тип stringValue) → `callback_message_id = "0"` (строка!)
- Is Callback? (Stats) проверяет `notEmpty` → `"0"` ≠ empty → TRUE
- Маршрут идёт в Edit Stats (Inline) с `message_id = "0"` → Telegram error 400 → тишина

**Фикс (01_Dispatcher):**
1. Template Engine: `let _cb_msg_id = ''` + оба fallback `|| ''` (было `|| 0`)
2. Prepare for 04: `callback_message_id = $json._cb_msg_id || ''` (было `|| 0`)
3. Early Typing Action — новая HTTP Request нода (fire-and-forget, dead-end), подключена параллельно от Telegram Trigger. Пользователь видит "typing..." через ~50ms вместо ~1с.

**Применено:** ✅ n8n API PUT (6XzdbQqpoUG0nsLe) — 48 нод, deactivate + activate

**Правило добавлено в CLAUDE.md:** `callback_message_id` fallback ОБЯЗАН быть `''`, НИКОГДА `0`. Строка `"0"` проходит `notEmpty` check → ломает все reply-кнопки.
