## Lessons learned

### Lesson 1: `is_idempotent_message` — claim, не check (см. PR #177)
INSERT...ON CONFLICT DO NOTHING — это атомарная операция «попытаться положить + узнать удалось ли». **Только один** caller должен вызывать его per (tid, mid). Defensive double-check в downstream — гарантированно блокирует real flow. Anti-pattern.

→ Cleanup KB: создать concept `claim-vs-check-idempotency-anti-pattern.md` (TODO для следующего агента).

### Lesson 2: transient Supabase pooler glitches ~30 сек
Уже 3 случая за последние 2 недели (10:00 MSK, 10:50 MSK сегодня + lesson runs 26103598563/26103611524 от 19.05). Smoke test 2-min окно даёт false positive. Fix: retry с свежим окном.

### Lesson 4: Sleep_checkin push «в 10:20» = корректный (timezone confusion)
Owner написал что sleep пуш пришёл в 10:20 после того как он залогировал хлеб в 10:19. По факту — он в `Europe/Madrid` (UTC+2), пуш отправлен в **09:20 Madrid** (= 07:20 UTC). Telegram клиент owner'а в MSK показывает timestamp 10:20 MSK = 09:20 Madrid. Mutex работает корректно.

→ KB-урок: добавить в [[concepts/release-protocol]] или новый concept `timezone-debugging-guide.md`. UI timestamps Telegram-клиента ≠ user_tz в БД.

### Lesson 5: Subagent для i18n × N langs — proven workflow
Pattern (повторили 3 раза):
1. Spawn copywriter subagent с full brief + KB-документы для tone reference
2. Subagent делает Part 1 (translations × N langs) + Part 2 (cultural review notes) + Part 3 (self-check)
3. Main agent генерирует SQL программно (Python script) — safer для Unicode/RTL/ZWNJ чем hand-writing
4. Preview для Номса в markdown × N langs (компактные таблицы)
5. После Номсова APPROVE/CHANGE — apply + DO $$ verify + commit + PR

ETA per session: ~30-60 мин. Quality: 5/5 critic per Sassy Sage rubric.
