## P0 hotfix — PR #166 (mig 322): «0 калорий» галлюцинация, Edit hidden, template polish

Owner+Noms ревью обнаружили 2 критических бага в LLM My Day:
1. **«Эй, ты на полпути! Ноль калорий, впереди целый день»** при 7 приёмах = 1637 ккал
2. **Кнопка [Исправить] пропала** на «Мой день» при ≥2 приёмах пищи

### Root cause #1: НЕ галлюцинация — broken payload
LLM **корректно** описывал то что получил: payload содержал `kcal_in=0, protein_g_in=0, fat_g_in=0, carbs_g_in=0, streak=0`. 

**Bug в `_normalise_day_summary`**: предполагал flat shape (`raw["total_kcal"]`), а реальная `get_day_summary` RPC возвращает **nested** структуру:
- макросы в `stats.{total_calories, total_protein, total_fat, total_carbs}`
- streak в `streak_current` (не `streak`)
- last_meal_at — derive из `meals[-1].meal_time`

5 из 12 полей silently возвращали 0.

**Fix**: новый helper `_s(...)` который читает из `stats.*` сначала, с flat fallback для forward-compat.

### Root cause #2: Edit visible_condition = 1
`stats_main` col 0 имел `visible_condition: (COUNT(meals today) = 1)`. Изначальная n8n логика: col 0 для 1 meal, col 1 (`cmd_show_meals`) для multi-meal picker. PR #165 удалил col 1 → multi-meal юзеры без кнопок.

**Fix** (mig 322): widen `= 1` → `>= 1`. Single [Исправить] cmd_edit_last (last meal). Full 2-stage picker — отдельный PR.

### Bonus #1: Template polish 💬 Noms → NOMS {emoji} × 13 langs (mig 322)
Backlog из PR #164 закрыт. Все 13 langs имели идентичный prefix `{icon_speech} Noms: "{insight_localized}"` → global replace на `NOMS {noms_emoji}: "{insight_localized}"`. Handler инжектит `noms_emoji`: cache hit → emotion's emoji, cache miss → 😏 default.

### Bonus #2: SAGE_MY_DAY_PROMPT logging
Добавлено info-логирование полного user_prompt (newlines escaped) перед каждым LLM call. Grep: `journalctl | grep SAGE_MY_DAY_PROMPT`.

### Bonus #3: Cache wipe (mig 322)
