## Noms v3 UX fixes — PR #163

### 1. mig 318 — Edit button bug на «📊 Мой день» (stats_main)
**Симптом** (юзер 12:37): «Кнопка [Исправить] в меню [📊 Мой день] не работает».

**Root cause:** `stats_main` row 0 имеет ДВЕ кнопки c одинаковым `text_key='buttons.edit_last'`. col 0 cb=`cmd_edit_last` работает. col 1 cb=`cmd_show_meals` — no-op stub от mig 211. Юзер нажимает «правую Исправить» → `editMessageText` 400 «message is not modified».

**Fix:** изменить col 1 → `[Удалить]` + `cmd_delete_last`. Pattern совпадает с food_log card `[Исправить][Удалить]`.

### 2. Sage timeout → всегда pre-baked fallback
**Симптом** (12:10:48): voice «кофе с молоком» → макросы залогированы → но `noms_comment` отсутствует.

**Root cause:** outer `asyncio.wait_for(sage_task, timeout=2.55s)` сработал ДО inner OpenAI client `timeout=2.5s`. 50ms headroom иногда тесный под jitter. Handler писал «skip comment».

**Fix:** новая public `services.sage.pick_food_log_fallback(ctx)` — обёртка над `_pick_pre_baked(ctx, "food_log_fallback")`. Handler в `except asyncio.TimeoutError` вызывает её sync'но → per-language pre-baked (mig 315: 13 langs × 5 variants). Карточка НИКОГДА не рендерится без Noms-comment.

### 3. Custom tg-emoji → unicode rollback
**Noms feedback:** «3D-эмодзи в инлайн-размере не читаются».

`EMOTION_TO_EMOJI: dict[str, str]` (плоский map). Убрал `<tg-emoji>` HTML и `💬`. Новый формат: `NOMS {emoji}: "text"`. Mapping: smirk→😏, proud→🤩 (было 😤), side-eye→🤨 (было smirk fallback), sage→🧘‍♂️.
