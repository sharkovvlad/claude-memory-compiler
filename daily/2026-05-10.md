### Session (22:00-22:40 MSK) — fix asymmetry: menu_v3 status normalization (PR #39 +1 commit)

**Симптом** (пользователь предъявил скрин в чате): юзер `417002669` в `status='edit_phenotype'` кликает reply «👤 Профиль» → бот: «⚠️ Что-то пошло не так. Попробуй ещё раз.». Второй клик — норм (status уже `'registered'`).

**Расследование:**
- Логи `journalctl noms-webhooks` около 19:16:54 UTC показали `[noms.handlers.menu_v3] ERROR: menu_v3 error code=error details={...meta..., 'bundle': 'dispatch_with_render', 'screen_id': 'profile_main', 'telegram_ui': {...полная kb...}}` — response **корректный** (profile_main отрендерен), но handler сигналит ошибку.
- Read `handlers/menu_v3.py:290`: `status = result.get("status") or "error"`. Response не содержит `status`, default = `"error"` → error envelope.
- Прочитал live SQL `dispatch_with_render` (`pg_get_functiondef`): для `v_user_status='edit_phenotype'` срабатывает onboarding-gate (mig 188) → `process_onboarding_input` → catch-all (L341-344): `UPDATE status='registered'` + `RETURN public.render_screen('profile_main')` БЕЗ обёртки `status='render'`.
- Прочитал `process_user_input` (L460): корректно оборачивает `RETURN v_render || jsonb_build_object('status', 'render')`. **Контракт пары различается:** `process_user_input` оборачивает, `process_onboarding_input` — нет.
- Прочитал `handlers/onboarding_v3.py:518-526`: **уже имеет нормализацию** «status=None + telegram_ui present → render». Комментарий явно фиксирует контракт.
- **Asymmetry handlers, не SQL contract bug.** `menu_v3` не имел нормализации, потому что до 10.05 в проде путь `menu_v3 → dispatch_with_render → onboarding-gate → process_onboarding_input` не было: legacy 04_Menu обслуживал phenotype-callbacks для `edit_phenotype` юзеров. После моего strip'а (21:47 MSK) — путь стал активным → bug проявился.

**Live репродукция** (savepoint):

**Альтернатива (отвергнута):** SQL fix mig 196 обернуть все 43 `RETURN public.render_screen(...)` в `process_onboarding_input`. Тяжёлая SQL surface, 4 из 43 — validation_error paths (требуют status='validation_error', не 'render'), риск семантического сдвига. Build script `scripts/_build_mig_196.py` запустился — нашёл 43 сайта вместо ожидаемых 10. Decision: Python fix (4 строки) вместо SQL fix (43 сайта).

```python

    status = "render"
else:

```
