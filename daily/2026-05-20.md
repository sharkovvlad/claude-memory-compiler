## Mig 285 — Streak food-log i18n + log_meal_transaction RPC fix

**Bug, replayed:** owner-observed на live tid=417002669 и 786301802 — после food log бот говорит «🏆 Стрик: 18 дн.» / «🏆 Стрик: 3 дн.» на RU юзерах. Англоязычный получит ту же RU-строку.

**Root cause (NLM-assisted):**
- `log_meal_transaction` RPC хардкодит RU PL/pgSQL прямо в SQL-функции:
  ```sql
  v_streak_msg := v_icon_streak || ' Стрик: ' || v_new_streak || ' дн.';
  v_streak_msg := '📅 Старт нового стрика!';
  ```
- Это **double violation:** no-hardcode + i18n broken (один lang для 13).
- Phase 4 UX-копирайтинг (mig 231 `ru_terminology_streak_to_seria`) audit'ил только `ui_translations` JSONB — не `pg_proc`. **Разные источники истины.** На surface food-log streak ключа в словаре вообще не существовало.

**Lesson (new general):** ВСЕ surface-генерирующие RPC должны быть audit'нуты на hardcoded localized strings отдельно. JSONB-словарь — лишь часть i18n-surface, RPC bodies — вторая часть. Без сoupled audit'а Phase 4 пропускает RPC composition surface'ы. См. KB pointer ниже.

2. **Upgrade `notebooklm-py` 0.3.4 → 0.4.1** в venv `NotebookLM+Claude/plugins/notebooklm-rag/.venv/`. В 0.4.x фиксы: «Per-domain cookie scoping on file upload» + «Server-state verification on timeout» (превентивно полезно при наших cookie-mixed warnings — `.google.es` duplicates). Smoke test после upgrade: auth ok, single upload ok.
3. **Bump delay в `noms-supabase-sync/SKILL.md`:** `--delay 2.0` → `--delay 3.0`. Файл живёт в `~/.claude/scheduled-tasks/noms-supabase-sync/SKILL.md` (вне git, change применится сразу).
4. **Retry/backoff в `nlm.py batch_add_sources`** (PR-style commit `4e91738` в локальном `NotebookLM+Claude` репо). 3 attempts с 5s/15s backoff, ловит маркеры: `SOURCE_ID`, `429`, `rate limit`, `timeout`, `500/502/503/504`, `temporarily`. Не-transient ошибки (file not found) fast-fail без ретрая. Happy-path и file-not-found code paths проверены.

**Defensive layers (3-level resilience):**

| Layer | Что делает | Trigger |
|---|---|---|
| L1: `--delay 3.0` | Reduce baseline throttle pressure | Always |
| L2: notebooklm-py 0.4.1 cookie scoping | Prevent mixed-domain cookie rejection | On upload |
| L3: retry с backoff | Recover from transient SOURCE_ID/429/5xx | On error |

Любая одна линия защиты могла бы предотвратить сегодняшний инцидент. Три вместе делают его маловероятным.

## Session 9 — P1 sprint #2: mig 292 BMR formula switch (Schofield/Molnar/Lührmann) — 2026-05-20 evening

**Закрыто:** P1.5 из [[calc-user-targets-roadmap]] — age-aware BMR formula switch. Mifflin-St Jeor систематически даёт ошибку у краевых возрастных групп (DR 2026-05-16 §Подростки, §Пожилые) — до 20-30% погрешности у подростков, +10-20% завышения у >75 (саркопения).

**Формулы (peer-reviewed):**
- **Henry 2005** Schofield-HW (PMID 16277825, SACN 2011 endorsed) для `<18 + BMI<30`:
  - Male: `15.6×W + 266×H + 299` (W=kg, H=м, kcal/day)
  - Female: `9.40×W + 249×H + 462`
- **Molnar 1995** (PMID 7562290, accuracy 87% per Acosta 2010 AJCN) для `<18 + BMI≥30`:
  - Male: `(50.0×W + 25.3×H_cm − 50.3×A + 26.9) / 4.184`
  - Female: `(51.2×W + 24.5×H_cm − 207.5×A + 1629.8) / 4.184`
- **Lührmann 2002** (PMID 12111047) для `>75`, no-height (elderly height shrinks):
  - Male: `935.4 + 11.95×W − 3.657×A`
  - Female: `757.1 + 11.95×W − 3.657×A`

**Surprise discovery** от research subagent'а: Henry 2005 публикует **ОДНУ** формулу для 10-18 на пол — НЕ sub-split 10-12/13-15/16-18, как мы изначально полагали в roadmap. Sub-split не существует ни в Henry 2005, ни в Schofield 1985 HW form (sub-split есть только в weight-only Schofield). Использую canonical Henry 2005 10-18 single block per gender (SACN 2011 endorsed).
