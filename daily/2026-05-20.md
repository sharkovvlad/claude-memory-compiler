## Mig 285 — Streak food-log i18n + log_meal_transaction RPC fix

**Bug, replayed:** owner-observed на live tid=417002669 и 786301802 — после food log бот говорит «🏆 Стрик: 18 дн.» / «🏆 Стрик: 3 дн.» на RU юзерах. Англоязычный получит ту же RU-строку.

**Root cause (NLM-assisted):**
- `log_meal_transaction` RPC хардкодит RU PL/pgSQL прямо в SQL-функции:
  ```sql
  v_streak_msg := v_icon_streak || ' Стрик: ' || v_new_streak || ' дн.';
  v_streak_msg := '📅 Старт нового стрика!';
  ```
- Это **double violation:** no-hardcode + i18n broken (один lang для 13).
- Phase 4 UX-копирайтинг (mig 231 `ru_terminology_streak_to_seria`) audit'ил только `ui_translations` JSONB — не `pg_proc`. **Разные источники истины.** На surface food-log streak ключа в словаре вообще не существовало.
- Surface не имел ключа в `ui_translations` потому что строка собиралась inline в SQL через конкатенацию. Phase 4 не мог его найти.

**Lesson (new general):** ВСЕ surface-генерирующие RPC должны быть audit'нуты на hardcoded localized strings отдельно. JSONB-словарь — лишь часть i18n-surface, RPC bodies — вторая часть. Без сoupled audit'а Phase 4 пропускает RPC composition surface'ы. См. KB pointer ниже.

**Решение (Variant A locked by owner):**
- 2 новых ключа: `gamification.streak_kept` + `gamification.streak_new_start` × 13 langs = 26 entries
- RPC rewrite: SELECT translation by `v_user.language_code` → REPLACE `{streak}` → return as `streak_message`
- Backward-compat: field `streak_message` сохранён (n8n `03_AI_Engine` не трогаем)
- Forward-prep: новый field `streak_event` enum (`kept | new_start | same_day | none`) — готовится к будущей миграции composition n8n→Python

**Pipeline (writer→critic с 6-й осью):**
- Critic axes (по запросу owner'а): naturalness, sassy_sage_tone, anti_shame (mandatory 5/5), brevity, **cross_language_comprehension** (NEW), cultural_fit
- Writer rating: 23/26 approved, 3 fix_needed (FA ZWNJ + IT Serie→Striscia ×2)

- Critic verifyед IT `striscia` canonical для celebration (glossary line 437: `striscia` для празднования, `serie` для нейтрального отчёта)
