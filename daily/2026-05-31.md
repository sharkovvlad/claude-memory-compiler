## mig 391 — «Назад» в онбординге: phenotype-квиз + экраны maternal/cycle (follow-up к mig 390)

**Что:** доделана отложенная часть mig 390 (back-навигация в онбординге).

**Контекст:** mig 390 (PR #246, смержен 30.05 вечером) вернул «Назад» на линейных шагах (пол→цель) + ветке maternal, но явно оставил вне scope: (1) back внутри phenotype-квиза во время онбординга, (2) кнопки «Назад» на самих экранах maternal/cycle.

**Сделано (RPC `process_onboarding_input`, 3 anchor-replace из live pg_get_functiondef):**
1. **Phenotype-квиз back раскрыт на онбординг.** Условие `cmd_back AND v_status = 'edit_phenotype'` → `IN ('edit_phenotype','registration_step_phenotype_quiz')`. q2→q1, q3→q2, q4→q3 работают общей nav_stack-логикой (она уже была корректна для обоих режимов — current screen = `nav_stack->>-1`). q1/NULL-ветка теперь режим-зависимая: retake → `my_plan_settings` (сохранено), онбординг → шаг до квиза (`goal_type='maintain'` → `edit_goal`, иначе → `edit_speed`).
2. **Cycle cmd_back** был re-render того же экрана → теперь `cycle → onboarding_maternal_status` (status `registration_step_maternal`; цикл достижим только из maternal "no").

**Кнопки (ui_screen_buttons):**
- q1-q4 (`edit_phenotype`,`phenotype_q2/3/4`) back: vis `u.status = 'edit_phenotype'` → `IN ('edit_phenotype','registration_step_phenotype_quiz')`. `phenotype_result` НЕ трогал (на result основное действие — Continue, вне scope).
- Добавлены back-кнопки (`buttons.back`/`cmd_back`) на `onboarding_maternal_status` (r3), `_trimester` (r2), `_lactation_type` (r2), `onboarding_cycle_question` (r2).

**Ключевой gotcha (nav_stack двухрежимный):** при входе в квиз в онбординге (из goal=maintain или speed-pick) `edit_phenotype` НЕ push'ится в nav_stack — push есть только при advance q1→q2 (`push_nav 'phenotype_q2'`) и в profile-retake entry (вне этого RPC). Значит на q1 в онбординге `nav_stack->>-1` ≠ 'edit_phenotype' → `v_current_q=NULL` → попадает в q1-ветку. Это и нужно: NULL трактуется как «не на конкретном вопросе» → выход к шагу до квиза. В retake же entry-handler push'ит 'edit_phenotype', поэтому q1 определяется корректно. Поэтому ветку q1 разделил по `v_status`, а не по nav.

**Применено:** LIVE через psycopg2 (anchor-replace, идемпотентность по маркеру `mig 391:`). mig HEAD → **391**.

**Верификация (psycopg2, синтетический user TID=-999999001, всё в ROLLBACK):**
- 12 RPC-сценариев переходов (onb q2→q1/q3→q2/q4→q3, q1→goal(maintain), q1→speed(lose/gain), retake q1→myplan, retake q3→q2, maternal→age, trimester→maternal, lactation→maternal, cycle→maternal) — все OK.
- 8 рендеров клавиатур: back-кнопки присутствуют где надо; на retake q1 `cmd_quiz_skip` отсутствует (не сломан).
