### Session: 2026-04-13 — Fix: Payment flickering + free user flow + Back button

#### Issue A: Мелькание (тройной editMessageText) + тройной Stars invoice
- Причина: `get_all_plan_prices` возвращает `RETURNS TABLE` с 3 строками → n8n HTTP Request создаёт 3 items → Build Plans Text/Method Selection/Extract Plan/Build Crypto — каждый запускался 3× → 3 editMessageText + 3 sendInvoice
- Фикс: 4 новых Code-ноды `Limit 1 (All Prices/Card/Stars/Crypto)` после каждого price HTTP Request. Код: `return [$input.first()];` — схлопывает 3 items в 1 item. Downstream Code-ноды по-прежнему используют `$('Get X Price').all()` для агрегации всех цен, но запускаются только 1 раз.

#### Issue B: Free user flow сломан
- Причина: `Get Subscription` (прямой REST-запрос к `user_subscriptions`) возвращал `[]` для free-пользователей → 0 n8n items → `Payment Router` не запускался вообще
- Фикс: 
  1. `Merge Data` → `Payment Router` напрямую (убрана зависимость от Get Subscription в главном потоке)
  2. `Payment Router[0,7]` → `Get Subscription` (moved to Plans-path only)
  3. `Get Subscription` теперь с `fullResponse: true` → всегда возвращает 1 item
  4. Новая нода `Extract Sub`: `const body = $input.item.json.body || []; return [{ json: body[0] || {} }]` — извлекает подписку из body, fallback `{}` для free-пользователей
  5. `Build Plans Text`: `$('Get Subscription')` → `$('Extract Sub')`, убран `Array.isArray()` check

#### Issue C: Кнопка "Назад" из "Ваша подписка"
- Причина: `Edit Progress (inline)` использовал `parse_mode: "Markdown"` — имена ботов/пользователей с `_` ломали Telegram Markdown parsing → HTTP 400 → silent fail; также URL с `=` префиксом без `{{ }}`; нет `onError`
- Фикс:
  1. `Build Progress Text (inline)`: `*...*` → `<b>...</b>`, `_..._` → `<i>...</i>`
  2. `Edit Progress (inline)`: `parse_mode: Markdown` → `HTML`; убран `=` из URL; добавлен `onError: continueRegularOutput`
