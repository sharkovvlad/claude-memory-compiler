# Daily 2026-06-06

## ~11:00 — fix(ambassador): кнопки «Поделиться кодом» и «Статистика» не работали (mig 471, PR #342)

**Проблема:** owner репортнул что на экране Банда/Прогресс кнопки [Поделиться кодом] и [Статистика] ни к чему не ведут.

**Root cause:** оба `ui_screen_buttons` на `friends_info` имели `meta={}`. `process_user_input` находил кнопку в БД (FOUND), но `target_screen=NULL` → `v_next_screen = v_current_screen` → ре-рендер того же экрана `friends_info` → Telegram видел «not modified» или silent no-op → пользователь ничего не замечал.

Это известный паттерн «Gotcha 4» в `[[concepts/headless-architecture]]` §meta.target_screen — только на этот раз кнопки были добавлены при миграции headless но meta не заполнили.

**Доп. контекст:**
- Кнопки видны только ambassadors/paid-referrers/trainers (visible_condition), поэтому массово не репортили — только owner с `ambassador_tier='active'`.
- `share_invite_link` работал (url_template:'share_invite' в meta) — именно поэтому не-ambassador пользователи не замечали проблем.

**Что применено LIVE (mig 471 applied):**

1. `share_ambassador_code` button: добавлен `meta = {"url_template": "share_ambassador_code"}` → теперь URL-кнопка открывает Telegram share-диалог с текстом «Используй мой код NOMS417002669 для 10% скидки на NOMS Premium!»

2. `cmd_ambassador_stats` button: добавлен `meta = {"target_screen": "ambassador_stats"}` → теперь навигирует на новый экран

3. Новый экран `ambassador_stats`:
   - `business_data_rpc = 'get_friends_info_rpc'` (те же данные)
   - `text_key = 'ambassador.stats_detail_text'` (новый ключ, без CTA-цитаты Nomsa → content отличается от friends_info → Telegram editMessageText принимает)
   - `back_screen_id_default = 'friends_info'`
   - кнопка cmd_back

4. `template_engine.py`: добавлен resolver `_resolve_share_ambassador_code_url` → строит t.me/share/url с NOMS{telegram_id}

5. Новый translation key `ambassador.stats_detail_text` × 13 языков: `📊 <b>{title_resolved}</b>\n\n{body_formatted}\n\n{link_block}`
