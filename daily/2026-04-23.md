**Root causes:**
1. Template_vars двойной nesting: RPC возвращал `template_vars` key → render_screen оборачивал в `telegram_ui.template_vars` → получалось `template_vars.template_vars.calories_consumed` → Dumb Renderer не находил flat `{var}`.

3. Translation placeholder `{tr:report.unit_kcal}` внутри composite {var} — Dumb Renderer делает 3 sequential pass'а (`{tr:...}` → `{{const}}` → `{var}`), и {tr:} внутри {var} substitution **не резолвится** (pass уже прошёл).

**Fix (migration 124, 408 строк):**
- RPC `get_daily_stats_rpc(bigint) → jsonb` с FLAT top-level fields (19 placeholder-ов + `stats`/`meals`/`targets`)
- Template stats.main_text × 13 langs, единый layout с `{tr:...}` / `{{const}}` / `{var}`
- Светофоры p/f/c_status: CASE по app_constants.macro_threshold_* (30/85/110%)

- meals_list_formatted через `string_agg` + `row_number()` с literal-resolved `unit_kcal` per user's language (fallback → en)
- no_meals_yet резолвится per user's language (empty-state case)
- DROP FUNCTION get_stats_business_data
- 1 кнопка `[✏️ Исправить]` с `visible_condition` на meals > 0
