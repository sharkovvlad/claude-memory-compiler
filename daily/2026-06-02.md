## Sage ToV-аудит по live-логам owner → патч промптов (PR #283, НЕ merged)

**Что:** ToV-ревизия тона Sage по реальным логам owner `417002669` (01–02.06: food-log
реакции + «Мой день»). 7 точечных правок промптов в `services/sage.py`
(`_DEFAULT_SYSTEM_PROMPT_EN` + `_DEFAULT_SYSTEM_PROMPT_MY_DAY_EN`). Текст-только, без
миграций. PR #283 открыт, **на merge у тимлида** (агент из worktree не деплоит).

**Зачем (находки):**
- **Формула из 3 рефлексов на каждый лог** → предсказуемость убивает «живого собеседника»:
  (1) остроумный заход → (2) биохим-штрих (инсулин/липиды) → (3) хедж «завтра сбалансируем?».
- **Баг правды:** реакция на кашу заявила «зашкаливает по углеводам» при `C18%`. Payload
  food-log содержит строку `Today so far ... C%` (`_build_day_context_lines`, sage.py:764) —
  модель проигнорировала. У food-log НЕ было гардрейла «не противоречь итогам дня», хотя у
  My Day он есть (sage.py:1170). Это был потерянный близнец.
- **Алармизм в халате:** «липаза на грани» прошёл, т.к. запрет покрывал только liver/pancreas,
  а не «фермент на грани».
- **Хедж на чистой еде:** зелёный салат 241 ккал → «не забудем про баланс» (балансировать нечего).
- **Предписания следующей еды:** «не забудь про углеводы позже» (тот же жанр, что забаненные
  «пей воду / добавь клетчатку»).
- **Судящее «переборщил» + закрытие дня в обед** в My Day (guardrail #3 гнал «завтра» на 50%-дне).
- **Просторечие в over_target** («жиров просто завались»).

**Правки:**
1. Характер (оба промпта): `wear knowledge lightly` — биохимия не флекс.
2. MACRO OVERFLOW: обобщён запрет алармизма (любой «орган/фермент на грани/maxed out»);
   биохимия = OPTIONAL rare spice с ротацией, не рефлекс; balance-хинт только при реальном переливе.
3. food-log forbidden: + запрет предписаний следующей еды.
4. DAY CONTEXT пп.2-3: запрет противоречить `Today so far %%` (18% = LOW, не «высоко»);
   «заслужи оговорку» (чистая еда = чистая реакция); эндокринология demoted в опцию.
5. My Day guardrail #3: не закрывать открытый день в обед; убрано «переборщил/перебрал».
6. My Day over_target: планка слога — острая экономная строка, бан просторечия И вычурного «пурпура».

**Намеренно НЕ тронул:** биохимия не выпилена (30% характера Sage) — снята с позиции дефолта
в редкую специю с ротацией. Owner согласовал направление «острее, не пышнее» (отверг встречный
рецепт другого ИИ-агента «больше биохимии / пышные метафоры» — тот ломал лимит 250 символов и
создавал псевдоумные нечитаемые метафоры типа «демографический взрыв жиров»).

**⚠️ После merge+deploy ОБЯЗАТЕЛЬНО:** insight «Мой день» кэшируется (4ч/фазе) — промпт-правки
не видны, пока не сбросить: `UPDATE users SET my_day_insight_at=NULL WHERE telegram_id=417002669;`

## Adaptive modifiers premium-gate bug + my_plan modifier-aware (mig 424, PR #284)

**Запрос owner'а (тест 417002669):** сон/стресс рисуют кнопки, но КБЖУ не пересчитывается. Плюс: «экран Подписка показывает Премиум до 8 авг, а агент сказал not-premium — как так?». Плюс: проверить научность сплита БЖУ (Б139/Ж79/У344).

**Расследование (live psycopg2 + KB):**
1. `apply_daily_modifier`: всегда пишет qualitative в `daily_metrics` (free), считает дельты, потом premium-gate. Гейт стоял на `user_has_renewable_sub(tid)` → требует `cancelled_at IS NULL`.
2. У 417002669 активная `quarterly` подписка `status='active'`, `expires_at=2026-08-08`, **`cancelled_at=2026-05-25`** (отменил автопродление), `payment_method='wallet'`. → `renewable=False` → гейт трактует как free → 12 строк `daily_modifiers` за 3 дня все `applied=FALSE/premium_required/deltas 0`. КБЖУ не двигалось нигде.
3. **Экран НЕ врал:** `get_subscription_business_data` считает подписку по `status='active'` (cancelled показывает «Доступ до {date}»). Это by design. Сбил с толку мой первый зонд `user_has_renewable_sub`.
4. Корень = **renewable ≠ entitlement.** KB `adaptive-modifiers-architecture:235` прямо называл `user_has_renewable_sub` «canonical Premium check» — ложь, исправлена.

**Fix (mig 424, генерация из live `pg_get_functiondef`):**
- **A:** new `user_has_active_premium(tid)` = `status='active' AND payment_method!='trial' AND expires_at>now()`. Свапнут в `apply_daily_modifier` + `get_daily_stats_rpc` (mig 348 strip gate). `user_has_renewable_sub` НЕ тронут (нужен для button visible_condition resume/cancel).
- **B:** `get_my_plan_business_data` накладывает `compute_daily_modifier_stack` дельты (формула как `get_day_summary`), пересчитывает % из скорректированных грамм. Это экран куда ведёт кнопка (`next_on_submit=my_plan`). Non-premium→0→no regression.

**Verified LIVE (ROLLBACK txn для функционала):** `active_premium=True` (было renewable=False); `apply sleep/short`→`applied=true` белок+15%/угли−15%; my_plan 139/79/344→**160/79/292** ккал 2643; day_summary совпал. p95 VPS: my_plan 45ms, daily_stats 47ms.
**Гайка:** сегодняшние ранние таппинги остались `applied=FALSE` — owner должен нажать заново чтобы увидеть.

**Наука БЖУ (нутрициолог-ревью):** Жир 79г = floor `0.8 г/кг` (обоснован). Белок 139г = `protein_g_per_kg_cardio=1.4` × 99кг → **занижен** для дефицита (норма 1.6–2.2 г/кг). Углеводы 344г (52%) = остаток → раздуты как следствие низкого белка. → **Fix C** (goal-aware protein floor для lose) спроектирован, НЕ реализован, design в `personalized-macro-split.md §PROPOSAL`.
