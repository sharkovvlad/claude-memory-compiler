### Session — Streak 7 days: verdict on current mp4 + improved prompt template + KB checklist for prompt-writing agents

**Контекст:** пользователь получил от Nano Banana 6-секундное видео для сценария «Стрик 7 дней» (Номс с пиксельными очками и пламенем). Спросил подходит ли + как улучшить промпт.

**Анализ текущего mp4:** NOT suitable. Проблемы:

1. **Перспективная камера с плиточным полом** — пол semi-transparent (alpha-blended с magenta) → создаст pink halo вниз, который **математически unfixable** (см. KB 7.5). Та же проблема что была со Streak 3 days "splash в желейную лужу".

**Вывод:** ни одна секунда видео не годится. Нужна перегенерация.

**Композировал новый промпт** для Nano Banana image-to-video:

- 6 sec duration (Nano Banana minimum)
- Seamless loop (final frame = first)
- Locked camera, orthographic front-facing
- **Periodic motions only**: idle vertical sine bounce, flame pulsation (8-bit), sunglasses glow flicker, binary code Matrix stream
- **Hands stay raised** (V-pose), no waving/rotation
- Explicit negatives: no floor, no reflective tiles, no dancing, no walking, no perspective, no watermark, no scene transitions

**Вторая итерация — ping-pong 2.8-4.3с (221 KB, финал):**
- Точка зажигания идентифицирована на t≈3.6-3.8с (пламя начинает вырываться из факела по сторонам, руки начинают подниматься).
- Single-pass с любым окном захватывающим factor → имел бы резкий seam (factor pose ↔ peak pose). Решение — **ping-pong** (как `streak_3days` v5).
- Forward 2.8→4.3 (1.5с): factor → climax. Reverse 4.3→2.8: «пламя втягивается обратно в факел».
- Семантика loop: «факел → разгорается → втянулось → факел». Логично для темы стрика (циклически нарастающее).
- Скрипт `make_sticker.py` не имеет встроенного `--pingpong` флага — собирал источник вручную через `ffmpeg ss=2.8 t=1.5 → reverse → concat` (рецепт handover §9). Затем pipeline на pre-built ping-pong mp4 с `--start 0 --duration 3`.

2. **`delete_only` strategy для multi-strategy envelope**: когда финальный экран — sticker + send_new (sticker через отдельный API), `delete_and_send_new` не подходит. Pattern: prepend `delete_only(ctx.last_bot_message_id)` через helper.
3. **NO_INDICATOR_STATUSES расширять при каждом новом reply-keyboard входе**: free-form text проходит через индикатор как «food» если статус не в whitelist.
4. **Не путать typing-indicator с n8n AI response**: тимлид принял «👀 Так-так, что тут у нас...» за AI Engine output, но это `telegram_proxy.maybe_send_indicator` (293ms fire-and-forget). Источник текста — `app_constants` через `_get_analyzing_phrases(lang)`.
